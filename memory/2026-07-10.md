3. VBScript launcher - No CDP binding
4. Self-elevating batch file - PowerShell parsing issues
5. Task Scheduler with HIGHEST privileges - Ready but requires admin

**Root Cause:** Windows security stack prevents programmatic control of Chrome CDP. Requires manual admin elevation.

### OpenClaw CLI Limitation Discovered
**Commands that work:**
- `openclaw browser open [URL]` ✅
- `openclaw browser start/stop` ✅
- `openclaw browser screenshot` ✅

**Commands that fail:**
- `openclaw browser snapshot [ref]` → "Too many arguments"
- `openclaw browser evaluate --fn "..."` → "Too many arguments"
- `openclaw browser type [ref] "text"` → "Too many arguments"

**CLI bug:** Argument parsing splits on any spaces regardless of quoting.
