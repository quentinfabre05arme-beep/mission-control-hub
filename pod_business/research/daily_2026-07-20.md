### Issue Details

| Field | Value |
|-------|-------|
| **Error** | 401 Unauthorized |
| **Token Status** | Valid (expires 2027-07-14) |
| **Token Scopes** | Full permissions granted |
| **Last Successful** | Never (new setup) |
| **Days Blocked** | 8 |

### Token Analysis
- **Issued:** July 14, 2026 17:31 UTC
- **Expires:** July 14, 2027 17:31 UTC
- **Now:** July 20, 2026 08:54 UTC
- **Expired:** No

### Hypotheses (Why 401 Despite Valid Token)

1. **Shop ID Mismatch** — Token may be for different shop than configured
2. **Scope Issue** — Token scopes may not include required endpoints
3. **Rate Limiting** — IP-based or account-based throttling
4. **Account Verification** — Printify account not fully verified
5. **Token Revocation** — Token silently revoked for security

### Required Actions (User)

1. **Log into printify.com**
2. **Go to:** Account → API Tokens
3. **Revoke** existing token
4. **Create new token** with ALL scopes:
   - `shops.manage`, `shops.read`
   - `products.read`, `products.write`
   - `orders.read`, `orders.write`
   - `catalog.read`
   - `uploads.read`, `uploads.write`
5. **Copy new token** to clipboard
6. **Update** `pod_business/.env` file:
   ```
   PRINTIFY_API_KEY=your_new_token_here
   PRINTIFY_SHOP_ID=your_shop_id_here
   ```
7. **Test** with: `node check_token.js`
