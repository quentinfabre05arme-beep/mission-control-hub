## IMPROVEMENT MECHANISMS

### Mechanism 1: Pattern Library

**Location:** `improvement_log/patterns/`

**Types:**
- `intent_patterns.json` — How to understand asks
- `solution_patterns.json` — Proven solution templates
- `failure_patterns.json` — Known failure modes and fixes
- `tool_patterns.json` — Optimal tool sequences

**Auto-update:** Every mission adds to library

### Mechanism 2: Skill Evolution

**Location:** `skills/`

**Process:**
1. Identify gap (can't do X well)
2. Research best practices
3. Build minimal working version
4. Test on real missions
5. Refine based on outcomes
6. Promote to standard skill

### Mechanism 3: Template Refinement

**Location:** `templates/`

**Types:**
- Mission planning templates
- AAR templates
- Report templates
- Code structure templates

**Update rule:** After every use, if better way found, update immediately

### Mechanism 4: Self-Measurement

**Metrics tracked:**
- Tool calls per outcome
- Time to completion
- Error rate by mission type
- User satisfaction (inferred from friction)
- Learning velocity (lessons extracted)

**Dashboard:** `improvement_log/metrics.json` updated daily
